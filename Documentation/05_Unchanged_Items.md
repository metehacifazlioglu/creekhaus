# Intentionally Unchanged — CreekHaus Prototype v2

The Finalization Guide was explicit: *refine, don't redesign.* This is the list of things that were deliberately left alone, so it's clear they were considered and kept — not overlooked.

---

## Visual identity — preserved exactly
- **Color palette** — the forest green / warm cream / sage / terracotta-orange system is unchanged. Hexes were sampled from the v1 mockups so the rebuild matches rather than approximates.
- **Typography pairing** — elegant serif for headings, clean sans for body. (Specified as Playfair Display + Inter for the handoff.)
- **Card-based layout** — rounded cards, soft shadows, generous spacing. Unchanged.
- **Photography-led home headers** — the immersive photo hero on the Home/Dashboard screens. Preserved (gradient placeholder stands in for the final photo).
- **Branding** — CreekHaus wordmark + leaf mark, same placement and treatment.
- **Navigation style** — bottom tab bar with a center action button on mobile; left sidebar on web. The *style* is unchanged; only the *items* were made consistent.

## Screens kept essentially as-is
- **Manager → Shift Assignments.** This screen was already clear and well-structured in v1 (role cards with descriptions, assigned people, edit affordance). Kept; only type weight and nav consistency were touched.
- **Web → Team Members.** Table structure (Name / Role / Permission Level / Status) unchanged.
- **Web → Shifts day-table.** Deliberately *not* converted to tabs. The existing clickable day view was judged better; this was the recommended answer to the open question, not an oversight.

## Concepts retained from v1 (per the "Preserve" list)
- Voice recording capability
- Quick Guide
- Daily updates
- Incident reporting
- Shift checklists
- Manager oversight
- Web dashboard

## Things considered and deliberately *not* added
- **No tabs on web Shifts** (above).
- **No new features** beyond what the meeting notes justified. The combined voice/text field is a *consolidation* of existing v1 screens, not a new feature.
- **No SOP document system.** Quick Guide stays as links/cards by design.
- **No mid-service notifications/reminders.** The meeting was clear that checklists happen at open/close/transition only and leads "just know" — so no interval reminders were added.

---

If any item on this list should in fact change, it's a deliberate decision rather than a gap in the work — flag it and it's a quick adjustment.
